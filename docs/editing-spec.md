# 网页编辑功能产品规格说明书

> 版本：v1.0
> 编写日期：2026-05-16
> 基于：docs/editing-requirements.md（45 条需求）、docs/editing-feasibility.md（可行性调查）、docs/engineering-status.md（工程现状）

---

## 一、功能定位

网页编辑功能允许用户在 Markdown 预览页面中一键切换到编辑模式，直接在浏览器中修改 Markdown 源码并保存回文件，保存后预览区域自动刷新展示最新渲染结果。功能对标 GitHub 的 "Edit / Preview" 标签切换体验，解决用户在预览中发现问题后需要切换回外部编辑器修改的痛点，覆盖高频微编辑场景。

**边界**：本功能仅支持 `.md` 文件的编辑与保存，不做新建/删除/重命名等文件管理，不做实时协同编辑，不做语法自动补全或格式化。

## 二、用户流程

### 2.1 主流程（Happy Path）

**步骤 1**：用户正在浏览器中浏览某个 `.md` 文件的渲染预览页面，在页面显眼位置看到一个「编辑」按钮。

**步骤 2**：用户点击「编辑」按钮，页面从预览模式切换为编辑模式：
- 原预览内容区域被一个编辑区域替代；
- 编辑区域内自动填入当前 Markdown 文件的原始源码文本（未渲染的 Markdown 内容，而非渲染后的 HTML）；
- 编辑区域占据足够大的空间，用户可舒适浏览和修改。

**步骤 3**：用户对 Markdown 文本进行所需修改（修正错别字、调整排版、更新链接等）。

**步骤 4**：用户点击「保存」按钮或使用键盘快捷键触发保存。

**步骤 5**：系统将编辑器中的完整内容覆盖写入磁盘上的原始文件：
- 保存成功后，页面显示一条短暂的「保存成功」提示（1--3 秒内自动消失）；
- 预览区域自动刷新，展示基于新文件内容的最新渲染结果；
- 页面从编辑模式退回预览模式。

**步骤 6**：用户继续浏览更新后的渲染内容。

### 2.2 取消流程

**步骤 1**：用户在编辑模式下进行了一些修改，但决定放弃这些修改。

**步骤 2**：用户点击「取消」按钮：
- 编辑器中的所有未保存修改被丢弃；
- 页面恢复到进入编辑模式之前的预览状态，展示原始文件内容的渲染结果。

### 2.3 异常流程

**场景 A：保存时网络故障或服务端异常**

用户点击「保存」后，由于网络中断或服务端错误导致保存失败：
- 系统显示明确的错误提示信息，告知用户「保存失败」及失败原因（使用用户可理解的语言）；
- 编辑器保持打开状态，用户已编辑的内容不丢失；
- 用户可以修复网络问题或等待服务恢复后重新尝试保存。

**场景 B：文件在外部被删除**

用户正在编辑某个文件时，该文件被外部程序删除：
- 用户点击「保存」时，系统提示文件已不存在，并建议用户退出编辑模式。

**场景 C：编辑期间外部修改文件**

用户在编辑模式下修改内容时，该文件被外部程序修改（触发热重载）：
- 编辑器保持打开状态，不自动关闭或刷新；
- 用户当前正在编辑的内容不丢失；
- 外部修改导致的热重载可能作用于其他已打开的浏览器标签页。

**场景 D：大文件加载**

用户尝试编辑一个内容极大的文件（超过系统配置的上限）：
- 系统在进入编辑模式之前告知用户文件过大，不加载编辑界面；
- 用户不会在编辑完成后才因文件过大致保存失败。

**场景 E：保存过程意外中断**

用户在保存过程中，系统进程意外崩溃或磁盘写入被中断：
- 原始文件保持完整，不会出现部分写入或文件损坏的情况。

## 三、界面规格

### 3.1 编辑入口

- **位置**：编辑入口按钮位于主内容区域的显眼位置，与页面整体布局协调。参考现有「主题切换」按钮的位置风格，可采用类似的固定位置控件。
- **样式**：按钮外形清晰，带有明确的文字标识「编辑」或含义等效的图标+文字组合，用户无需猜测其用途。
- **可见性规则**：
  - 仅在当前页面展示的是 `.md` 文件时才显示编辑入口；
  - 在非 Markdown 页面、空状态页面中，编辑入口不显示；
  - 在目录模式下，编辑入口的指向跟随侧边栏中当前高亮选中的文件变更。

### 3.2 编辑模式

- **布局变更**：
  - 原预览渲染的内容区域被一个编辑区域替代；
  - 侧边栏（目录模式时）和 TOC（目录）保持原有位置与状态不变；
  - 页面整体布局结构不发生剧烈变动，避免用户迷失上下文。
- **视觉区分**：
  - 编辑模式下的背景、边框或整体色调与预览模式有明显区别，使用户清楚感知当前处于编辑状态而非查看状态；
  - 编辑区域的视觉风格与页面整体设计语言一致。

### 3.3 编辑区域

- **尺寸**：编辑区域占据主要内容区的全高，提供充足的纵向空间用于浏览和编辑长文档。
- **字体**：编辑区域使用等宽字体显示 Markdown 源码，确保代码块、表格、ASCII 图表等依赖对齐的内容保持原始格式。
- **配色**：编辑区域的配色方案自动跟随当前页面主题（浅色/深色模式）切换，与页面整体视觉风格协调一致。
- **文本操作**：支持标准文本编辑操作，包括光标移动、文本选择、剪切、复制、粘贴、撤销、重做。

## 四、交互规格

### 4.1 进入编辑

| 项目 | 描述 |
|------|------|
| 触发方式 | 点击「编辑」按钮 |
| 前置条件 | 当前页面为 `.md` 文件的预览页面 |
| 即时反馈 | 预览内容区域隐藏，编辑区域显示并自动填入 Markdown 原始源码 |
| 加载状态 | 若获取原始内容需要短暂等待，编辑区域应显示加载指示 |

### 4.2 编辑中

| 项目 | 描述 |
|------|------|
| 键盘导航 | 编辑模式下，原有的左右箭头键文章导航功能自动禁用，防止与编辑器光标移动冲突 |
| 语法高亮 | （第二阶段）编辑区域内的 Markdown 源码可带有语法着色，帮助用户辨识标题、链接、代码块等结构 |
| 主题同步 | （第二阶段）用户切换页面主题（浅色↔深色）时，编辑区域配色实时跟随变化 |
| 外部修改 | 编辑期间若文件被外部修改，编辑器保持打开且不丢失用户正在编辑的内容 |

### 4.3 保存

| 项目 | 描述 |
|------|------|
| 触发方式 | 点击「保存」按钮；或（第二阶段）使用键盘快捷键 Ctrl+S / Cmd+S |
| 保存范围 | 编辑器内的**全部内容**（全量覆盖写入，非增量或行级保存） |
| 保存限制 | 仅允许保存 `.md` 文件；文件路径受安全校验，防止越权访问 |
| 成功反馈 | 显示「保存成功」提示，1--3 秒内自动消失；页面自动退回预览模式并刷新渲染 |
| 失败反馈 | 显示「保存失败」提示，附带用户可理解的错误原因；编辑器保持打开，内容不丢失，用户可重试 |
| 数据安全 | 保存过程中如发生意外中断，原始文件内容不损坏（原子写入保障） |
| 热重载联动 | 保存成功后，所有已打开的浏览器标签页通过已有热重载机制自动刷新 |

### 4.4 取消/退出

| 项目 | 描述 |
|------|------|
| 触发方式 | 点击「取消」按钮 |
| 行为 | 编辑器关闭；所有未保存修改丢弃；页面恢复到进入编辑前的预览状态 |
| 离开确认 | （第二阶段）若编辑器内容与原始文件存在差异，执行取消操作或关闭浏览器标签页时，弹出确认对话框，要求用户明确确认放弃修改 |

### 4.5 键盘操作

| 键盘操作 | 生效阶段 | 行为 |
|----------|---------|------|
| Ctrl+S / Cmd+S | 第二阶段 | 在编辑器获得焦点时触发保存（不触发浏览器默认的「保存网页」行为） |
| Esc | 第二阶段 | 触发取消操作（若存在未保存修改，弹出确认对话框） |
| ← / → 箭头键 | 编辑模式下 | 在编辑器内移动光标（不触发文章导航切换） |
| Ctrl+Z / Cmd+Z | 任意阶段 | 在编辑器内撤销文本修改 |
| Ctrl+Shift+Z / Cmd+Shift+Z | 任意阶段 | 在编辑器内重做文本修改 |

## 五、验收标准 (AC)

### 5.1 核心功能验收（MVP）—— AC-MVP

---

**AC-MVP-01：编辑入口可见性与切换**

- **覆盖需求**：FR-01、FR-02、FR-03
- **前置条件**：用户在浏览器中打开一个 `.md` 文件的预览页面
- **操作**：观察页面内容区域的指定位置
- **预期结果**：
  - 页面显示一个含义清晰的编辑入口控件（如「编辑」按钮）；
  - 将鼠标悬停在按钮上时，光标样式变为可点击状态；
  - 同一按钮在非 Markdown 页面（如空状态页）不显示；
  - 在目录模式下切换到侧边栏中另一个 `.md` 文件后，编辑入口的指向随之更新为该文件。

---

**AC-MVP-02：编辑模式加载原始内容**

- **覆盖需求**：FR-04、FR-05、FR-06
- **前置条件**：用户在 `.md` 文件预览页面，文件包含已知的 Markdown 标记（如 `# 标题`、`**粗体**`）
- **操作**：点击「编辑」按钮
- **预期结果**：
  - 预览渲染内容区域被编辑区域替代；
  - 编辑区域内显示的是 Markdown 原始源码（可见 `# 标题`、`**粗体**` 等标记字符），而非渲染后的 HTML；
  - 编辑区域占据足够的高度以支持长文档编辑；
  - 编辑区域的视觉风格（背景色、边框等）与预览模式有明显区别，用户可明确感知当前处于编辑状态。

---

**AC-MVP-03：编辑→保存→刷新完整闭环**

- **覆盖需求**：FR-10、FR-12、AC-01
- **前置条件**：用户进入编辑模式，对内容进行了可识别的修改（如在末尾添加一行新文字）
- **操作**：点击「保存」按钮
- **预期结果**：
  - 页面显示短暂的成功提示（1--3 秒内自动消失）；
  - 页面自动从编辑模式退回预览模式；
  - 预览区域刷新，展示包含用户新增内容的最新渲染结果；
  - 用外部文本编辑器打开同一个 `.md` 文件，可验证修改内容已被写入磁盘文件。

---

**AC-MVP-04：取消编辑并丢弃修改**

- **覆盖需求**：FR-14
- **前置条件**：用户进入编辑模式，对内容进行了可识别的修改
- **操作**：点击「取消」按钮
- **预期结果**：
  - 编辑器关闭，所有未保存的修改被丢弃；
  - 页面恢复到进入编辑模式之前的预览状态，展示原始文件内容的渲染结果；
  - 用外部文本编辑器打开该 `.md` 文件，确认文件内容未被修改。

---

**AC-MVP-05：路径穿越防护**

- **覆盖需求**：NFR-01、AC-02
- **前置条件**：系统以目录模式运行，服务目录为 `/home/user/docs`
- **操作**：尝试通过构造非常规请求，将内容保存到服务目录之外的路径（如试图越权写入上级目录或系统目录中的文件）
- **预期结果**：
  - 系统拒绝写入请求；
  - 页面显示「保存失败」提示（不暴露内部路径信息）；
  - 目标路径下没有文件被创建或修改。

---

**AC-MVP-06：文件类型限制**

- **覆盖需求**：NFR-02、FR-16、AC-02
- **前置条件**：用户正在预览一个 `.md` 文件的渲染结果
- **操作**：尝试通过构造非常规请求，将内容保存到 `.html`、`.txt` 或其他非 `.md` 扩展名的文件
- **预期结果**：
  - 系统拒绝该保存请求；
  - 页面显示「保存失败」提示。

---

**AC-MVP-07：内容大小限制**

- **覆盖需求**：NFR-03
- **前置条件**：用户进入编辑模式
- **操作**：尝试保存一份内容大小超过系统设定上限（如 10 MB）的 Markdown 内容
- **预期结果**：
  - 系统拒绝保存请求；
  - 页面显示大小超限的提示信息；
  - 原始文件未被修改。

---

**AC-MVP-08：原子写入——中断时文件不损坏**

- **覆盖需求**：NFR-04
- **前置条件**：用户进入编辑模式，对内容进行了修改；原始文件内容已知为 A
- **操作**：在保存过程中，模拟写入过程意外中断（如磁盘空间在写入中途耗尽、或系统进程在写入中途崩溃）
- **预期结果**：
  - 原始文件内容保持完整为 A，未出现部分写入（内容为 A 的前半部分 + 新内容的碎片）的情况；
  - 系统能正常恢复并继续服务（如重启后文件可正常读取）。

---

**AC-MVP-09：保存失败——网络或服务端异常**

- **覆盖需求**：EC-04、EC-05、AC-04
- **前置条件**：用户在编辑模式下对内容进行了修改
- **操作**：在保存过程中模拟以下两种异常情况之一：
  - 网络断开/超时；
  - 服务端返回错误（如磁盘已满、权限不足）
- **预期结果**：
  - 页面显示清晰的「保存失败」提示信息，说明失败原因（使用用户可理解的语言，不暴露技术细节）；
  - 编辑器保持打开状态，用户已编辑的内容不丢失；
  - 用户可以修复问题或等待恢复后重新尝试保存。

---

### 5.2 质量验收—— AC-QUAL

---

**AC-QUAL-01：标准文本编辑操作**

- **覆盖需求**：FR-07
- **前置条件**：用户处于编辑模式，编辑区域内有内容
- **操作**：分别验证以下操作：
  - 使用鼠标或键盘方向键在文本中移动光标；
  - 拖拽鼠标选中一段文本；
  - 使用 Ctrl+C / Cmd+C 复制选中文本；
  - 使用 Ctrl+X / Cmd+X 剪切选中文本；
  - 使用 Ctrl+V / Cmd+V 粘贴内容；
  - 使用 Ctrl+Z / Cmd+Z 撤销上一步修改；
  - 使用 Ctrl+Shift+Z / Cmd+Shift+Z 重做已撤销的修改
- **预期结果**：所有操作行为与用户在常见文本编辑场景中的预期一致，无异常。

---

**AC-QUAL-02：热重载兼容性**

- **覆盖需求**：NFR-06、AC-03
- **前置条件**：同时打开两个浏览器标签页，均预览同一个 `.md` 文件
- **操作**：在标签页 A 中进入编辑模式，修改内容并保存
- **预期结果**：
  - 标签页 A 预览自动刷新，展示最新渲染结果；
  - 标签页 B 也自动刷新（通过已有热重载机制），展示最新渲染结果；
  - 两个标签页的显示内容一致。

---

**AC-QUAL-03：键盘导航在编辑模式下禁用**

- **覆盖需求**：NFR-07、AC-03
- **前置条件**：在目录模式下，当前文件存在上一篇文章和下一篇文章
- **操作**：
  - 在编辑模式下，在编辑区域内按键盘左箭头（←）和右箭头（→）
- **预期结果**：
  - 左/右箭头在编辑区域内正常移动光标，不触发文章导航切换；
  - 退出编辑模式回到预览模式后，再次按左/右箭头键，文章导航功能恢复正常。

---

**AC-QUAL-04：主题切换与样式隔离**

- **覆盖需求**：NFR-08、AC-03
- **前置条件**：用户处于编辑模式
- **操作**：切换页面主题（浅色 ↔ 深色），可通过主题切换按钮或系统级配色偏好变更触发
- **预期结果**：
  - 页面整体主题切换正常，预览区域、侧边栏、TOC 等所有元素配色均正确切换；
  - 编辑区域及其关联控件（保存/取消按钮等）同步切换配色，无样式错乱或对比度问题；
  - 再次切换回原主题后，所有元素恢复正常。

---

**AC-QUAL-05：桌面端与平板端布局**

- **覆盖需求**：NFR-09、AC-05
- **前置条件**：使用桌面级设备，浏览器窗口宽度 >= 1024 px
- **操作**：进入编辑模式
- **预期结果**：
  - 编辑区域占据主要内容区全高，无横向溢出或布局断裂；
  - 编辑/保存/取消按钮可见且可点击；
  - 侧边栏（目录模式时）和 TOC 布局正常；
  - 将窗口宽度调整为 >= 768 px 但 < 1024 px（平板级），编辑区域仍合理可用，无布局断裂。

---

**AC-QUAL-06：主流浏览器兼容**

- **覆盖需求**：AC-05
- **前置条件**：使用 Chrome、Firefox、Edge 三款浏览器的最新两个大版本
- **操作**：在各浏览器中分别执行完整编辑闭环（进入编辑 → 修改 → 保存 → 查看更新 → 取消）
- **预期结果**：所有浏览器中编辑入口、界面布局、编辑操作、保存/取消操作均正常，无功能缺失或视觉异常。

---

**AC-QUAL-07：两种服务模式**

- **覆盖需求**：SM-01、SM-02、SM-04
- **前置条件 A（单文件模式）**：系统以单文件模式运行，指定一个 `.md` 文件
- **操作 A**：在预览页面查看编辑入口 → 进入编辑 → 保存
- **预期结果 A**：编辑功能完全可用，编辑操作作用于命令行指定的唯一文件。
- **前置条件 B（目录模式）**：系统以目录模式运行，目录下包含多个 `.md` 文件
- **操作 B**：在侧边栏中点击文件 A → 进入编辑模式 → 修改 → 保存
- **预期结果 B**：
  - 保存后预览刷新，侧边栏高亮仍指向文件 A；
  - TOC 显示当前文章的目录结构；
  - 侧边栏的展开/折叠状态保持不变。

### 5.3 增强功能验收（第二阶段）—— AC-ENH

---

**AC-ENH-01：Ctrl+S / Cmd+S 保存快捷键**

- **覆盖需求**：FR-11、AC-06
- **前置条件**：用户处于编辑模式，编辑器获得焦点
- **操作**：按下 Ctrl+S (Windows/Linux) 或 Cmd+S (macOS)
- **预期结果**：
  - 触发保存行为，效果与点击「保存」按钮完全一致；
  - 浏览器**不**弹出原生的「保存网页」对话框。

---

**AC-ENH-02：未保存修改离开确认**

- **覆盖需求**：FR-15、AC-06
- **前置条件**：用户进入编辑模式并对内容进行了修改（与原始文件存在差异）
- **操作**：分别触发以下两种退出行为：
  - 点击「取消」按钮；
  - 尝试关闭浏览器标签页或导航到其他页面
- **预期结果**：
  - 弹出确认对话框，内容类似「你有未保存的修改，确定要放弃吗？」；
  - 用户点击「取消/继续编辑」后，保持在编辑模式，内容不丢失；
  - 用户点击「确定/放弃」后，正常退出并丢弃修改；
  - 若编辑器内容与原始文件无差异，直接退出不弹确认框。

---

**AC-ENH-03：编辑器配色跟随主题**

- **覆盖需求**：FR-08、AC-08
- **前置条件**：用户处于编辑模式，页面当前为浅色主题
- **操作**：切换至深色主题，再切换回浅色主题
- **预期结果**：
  - 切换到深色主题后，编辑器背景变为深色、文字变为浅色，与深色 GitHub 风格一致；
  - 切换回浅色主题后，编辑器背景变为浅色、文字变为深色；
  - 编辑器内的语法着色（若有）同步更新。

**AC-ENH-04：等宽字体显示**

- **覆盖需求**：FR-09
- **前置条件**：用户进入编辑模式，编辑内容包含表格、代码块等依赖等宽对齐的 Markdown 片段
- **操作**：观察编辑区域内文本的视觉呈现
- **预期结果**：
  - 编辑区域使用等宽字体渲染所有文本；
  - 表格的列对齐、代码块的缩进、ASCII 图表的对齐均保持正确；
  - 中英文混合场景下，等宽表现不出现明显错位。

**AC-ENH-05：保存后恢复浏览位置**

- **覆盖需求**：FR-13
- **前置条件**：用户在一个较长文档的中间位置浏览，然后进入编辑模式
- **操作**：在编辑器中修改某处内容（如修正一个错别字），点击保存
- **预期结果**：
  - 保存成功后页面退回预览模式并刷新；
  - 页面滚动位置恢复到编辑前用户浏览的位置附近（而非跳回页面顶部），提供连贯的阅读体验。

**AC-ENH-06：边界场景综合验收**

- **覆盖需求**：EC-01、EC-02、EC-03、EC-06、SM-03、NFR-10、AC-07
- **场景 1 —— 空文件编辑**：
  - **前置条件**：当前 `.md` 文件为空（0 字节）
  - **操作**：点击「编辑」按钮进入编辑模式
  - **预期结果**：编辑区域正常显示为一个空白编辑区，用户可输入内容并成功保存。
- **场景 2 —— 大文件警告**：
  - **前置条件**：当前 `.md` 文件内容大小超过系统设定上限
  - **操作**：点击「编辑」按钮
  - **预期结果**：系统在进入编辑前告知用户「文件过大，无法编辑」，不进入编辑模式。
- **场景 3 —— 编辑期间外部修改不丢数据**：
  - **前置条件**：用户正在编辑模式中，已修改了部分内容但未保存；另一个外部程序修改了同一文件
  - **操作**：等待热重载可能触发（若有影响），继续编辑并保存
  - **预期结果**：编辑器保持打开状态，用户之前编辑的内容不丢失；用户可继续修改并成功保存。
- **场景 4 —— 文件被外部删除后保存**：
  - **前置条件**：用户进入编辑模式后，原始文件被外部程序删除
  - **操作**：点击「保存」按钮
  - **预期结果**：显示提示信息告知用户「文件已不存在」，建议用户退出编辑模式；编辑器内容不丢失。
- **场景 5 —— 编辑中切换文件提示**：
  - **前置条件（目录模式）**：用户正在编辑文件 A，已做了未保存修改
  - **操作**：点击侧边栏中的文件 B
  - **预期结果**：弹出确认对话框，提示用户选择「保存并切换」「放弃并切换」或「继续编辑」。
- **场景 6 —— 用户可理解的错误提示**：
  - **前置条件**：模拟各种服务端异常（磁盘满、权限不足、路径无效等）
  - **操作**：尝试保存
  - **预期结果**：所有错误提示使用用户可理解的语言描述（如「磁盘空间不足，无法保存」而非「ENOSPC」），不暴露服务端内部路径、堆栈信息或技术细节。
