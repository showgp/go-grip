## 一、阻断问题（必须先修正才能开始实现）

| 编号 | 类别 | 问题 | 修正方案 |
|------|------|------|---------|
| **B1** | 路径安全 | 符号链接绕过：`rootDir` 内若有符号链接指向外部路径，`filepath.Abs()` + `HasPrefix` 检查会失效 | 在 Task 1.2 安全校验步骤 4 后增加 `filepath.EvalSymlinks()` 解析符号链接，再验证前缀 |
| **B2** | 热重载文档 | 原子写入触发路径描述错误：`os.Rename` 在 Linux 上产生 `IN_MOVED_TO` → `Create` 而非 `Write` 事件。`Rename` 处理器只移除 watch 不广播 | 更正文档描述为：原子写 → `fsnotify.Create` → debounce → Broadcast。标注需在 macOS(kqueue) 和 Windows 上单独验证 |

## 二、遗漏需求（完全未覆盖）

| 编号 | 需求 | 影响 | 建议 |
|------|------|------|------|
| **FR-13** | 预览刷新后恢复浏览滚动位置（P1） | `window.location.reload()` 整页刷新丢失滚动状态 | 增加 Phase 2 任务：保存前将 `scrollTop` 存入 sessionStorage，刷新后在页面加载时恢复 |
| **EC-06** | 文件被外部删除后保存——提示文件不存在（P1） | 用户编辑中途文件消失，保存时报泛型错误 | 在 Task 1.3 写入前增加 `os.Stat` 存在性检查，返回专用"文件已不存在"错误 |
| **SM-04** | 保存刷新后保持侧边栏展开/折叠和高亮状态（P0） | 目录模式下刷新后侧边栏状态重置 | 增加 Phase 1 任务：侧边栏状态写入 sessionStorage，页面加载时恢复（可复用现有模板） |
| **AC-05** | Chrome/Firefox/Edge 最近两个大版本的兼容性测试 | 未计划任何浏览器兼容测试 | 在 Phase 1 增加跨浏览器验证任务（轻量，因仅使用标准 HTML/CSS/JS） |

## 三、验收标准无法满足（6 条）

| 编号 | 验收标准 | 根因 | 修正 |
|------|---------|------|------|
| **AC-MVP-01** | 编辑入口可见性——按钮仅在 `.md` 页面显示 | Task 1.4 模板无条件渲染按钮，缺少前端隐藏逻辑 | 在 Task 1.5 增加：读取 `data-current-file` 扩展名，非 `.md` 时隐藏按钮 |
| **AC-MVP-03** | 保存成功后显示 1-3 秒 toast 提示 | Task 1.5 保存后"静默等待刷新"，无 toast；Task 2.4 的按钮反馈因全页刷新不可见 | 将 Task 2.4 的保存成功反馈前移至 Phase 1；前端先等保存响应返回再主动刷新，不再依赖 WebSocket |
| **AC-MVP-07** | 内容大小限制——拒绝超限保存 + 编辑前预检 | 实现任务被推迟到 Phase 3，且缺少"进入编辑前预检"的前端逻辑 | 将大小限制从 Phase 3 移至 Phase 1 Task 1.2；在 Task 1.5 进入编辑前检查文件大小，超限则弹窗拒绝 |
| **AC-QUAL-05** | 平板端（768px-1024px）布局可用 | Task 1.6 仅"最小高度 300px"，无媒体查询 | 在 Task 1.6 增加：`@media (min-width:768px) and (max-width:1023px)` 平板适配规则 |
| **AC-ENH-05** | 保存后恢复浏览位置 | 全页刷新丢失位置 | 同 FR-13 修正方案 |
| **AC-ENH-06** | 边界场景综合验收（大文件预检、文件切换确认） | 2/6 子场景完全缺失 | 补充 Task 2.2：大文件预检弹窗；补充 Task 1.5：侧边栏文件切换拦截（编辑未保存时弹出三选项对话框） |
